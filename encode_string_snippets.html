Multi-purpose application for encryption tools



The purpose of this application is to replicate encoding functionality of https://www.base64encode.org/enc/map to understand the mapping of encoding methods. And, to serve as a method to generate private and public data encryption keys, such that one can place the keys manually in GitHub Secrets. Manually storing the private and public data encryption keys on the GitHub server, using GitHub Secrets, is a secure way to encode and decode data while protecting user information.

Generate a pair of private and public data encryption keys in JSON Web Key format, using the Rivest–Shamir–Adleman algorithm https://www.w3.org/TR/WebCryptoAPI/#rsa-oaep-params:

Copy the public and private JSON Web Keys and manually put them in GitHub Secrets, the public JSON Web Key is used to encode data and the private JSON Web Key is used to decode data. Use these keys on GitHub Actions, GitHub Secrets are only accessable from GitHub Actions, to encode and decode data securely! For more information about how to use these keys, please see the mozilla developer page at https://developer.mozilla.org/en-US/docs/Web/API/SubtleCrypto/importKey. The algorithm settings used to generate these keys were: {name: "RSA-OAEP", modulusLength: 2048, publicExponent: new Uint8Array([0x01, 0x00, 0x01]), hash: {name: "SHA-256"}}, true, ["encrypt", "decrypt"]. These settings are important to know for importing the key in GitHub Actions.
